# Claude Knowledge Graph

[![MIT License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![npm](https://img.shields.io/badge/npm-%40yaz231%2Fclaude--knowledge--graph-red)](https://www.npmjs.com/package/@yaz231/claude-knowledge-graph)

**One installer that wires CodeGraph symbol intelligence + persistent CLAUDE.md project memory into every Claude Code session.**

## Quick Start

```bash
npx @yaz231/claude-knowledge-graph
```

Or install globally:

```bash
npm install -g @yaz231/claude-knowledge-graph
ckg
```

## What It Does

### Layer 1 — CodeGraph (Symbol Intelligence)

[CodeGraph](https://github.com/colbymchenry/codegraph) pre-indexes your codebase into a SQLite symbol graph, then exposes it as an MCP server. Claude Code can query it to look up function definitions, find callers and callees, and assess the blast radius of a change — without reading every file from scratch each session.

CKG installs CodeGraph as a peer dependency, registers its MCP server in `~/.claude.json`, and runs `codegraph init` on any project you initialize.

### Layer 2 — CLAUDE.md Auto-Generation (Project Memory)

Every directory in your project gets a `CLAUDE.md` file generated by Anthropic's API. These files capture:
- What the directory contains and its purpose
- Key files and their responsibilities
- Conventions, patterns, and rules Claude should follow
- A running changelog of session summaries

Two Claude Code hooks keep this memory alive automatically:
- **PostToolUse** — after every file write or edit, the affected directory's `CLAUDE.md` is updated
- **Stop** — at the end of every session, a changelog summary is appended to the root `CLAUDE.md`

### How They Work Together

When you open a project with CKG initialized, Claude Code can:
1. Read `CLAUDE.md` files for narrative context (architecture, conventions, history)
2. Query CodeGraph for precise symbol lookups without reading entire files
3. Automatically keep both systems up to date as you work

This dramatically reduces the time Claude spends re-discovering project structure across sessions.

## CLI Reference

| Command | Description |
|---|---|
| `ckg` | Run the interactive installer |
| `ckg install` | Alias for the installer |
| `ckg init [path]` | Initialize CKG in a project (CodeGraph index + CLAUDE.md crawl) |
| `ckg uninit [path]` | Remove CKG from a project |
| `ckg status [path]` | Show CodeGraph status + CLAUDE.md coverage |
| `ckg sync [path]` | Re-index CodeGraph + regenerate stale CLAUDE.md files |

## Configuration

Config is stored in `~/.ckg/config.json` (override with `CKG_CONFIG_DIR` env var):

| Key | Default | Description |
|---|---|---|
| `anthropic_api_key` | `""` | Anthropic API key for CLAUDE.md generation |
| `python_path` | `"python3"` | Python interpreter used to run hooks |
| `hook_model` | `"claude-haiku-4-5-20251001"` | Model for fast hook updates (PostToolUse/Stop) |
| `init_model` | `"claude-sonnet-4-6"` | Model for initial CLAUDE.md generation (`ckg init`) |

### Files Written by the Installer

| Path | Purpose |
|---|---|
| `~/.ckg/config.json` | CKG configuration |
| `~/.ckg/scripts/init_claude_md.py` | CLAUDE.md crawler script |
| `~/.claude/hooks/post_tool_use.py` | PostToolUse hook |
| `~/.claude/hooks/stop.py` | Stop hook |
| `~/.claude/settings.json` | Hook registration (deep-merged) |
| `~/.claude.json` | CodeGraph MCP server registration (deep-merged) |
| `~/.claude/CLAUDE.md` | Global CKG instructions for Claude (appended) |

### Per-project Files

| Path | Purpose |
|---|---|
| `.codegraph/` | CodeGraph SQLite index (gitignored) |
| `CLAUDE.md` (each directory) | Narrative context for that directory |
| `~/.ckg/session-log.json` | In-session event log (cleared after Stop hook) |
| `~/.ckg/logs/hook-errors.log` | Hook error log |

## Requirements

- **Node.js** 18+
- **Python** 3.8+ with `anthropic` package (`pip install anthropic`)
- **Claude Code** installed and configured
- **Anthropic API key**
