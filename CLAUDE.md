# Claude Knowledge Graph — Root

## Project Overview
NPM package (`@yaz231/claude-knowledge-graph`) that installs CodeGraph symbol intelligence + auto-generated CLAUDE.md persistent memory into Claude Code sessions via a single CLI command (`npx @yaz231/claude-knowledge-graph` or `ckg`).

## Tech Stack
Node.js ≥18 · TypeScript 5 (strict, ES2020, commonjs) · Python 3.8+ (hooks & scripts) · Anthropic API (CLAUDE.md generation)

## Directory Map
- `src/` — TypeScript CLI source (commands: init, status, sync, uninit; installer, config, utils, MCP server)
- `dist/` — Compiled JS output (gitignored, built via `tsc`)
- `hooks/` — Python PostToolUse/Stop hooks for Claude Code (auto-update CLAUDE.md on file changes and session end)
- `scripts/` — Python init crawler (`init_claude_md.py`) for first-time CLAUDE.md generation

## Key Files
- `package.json` — CLI entrypoint (`ckg` → `dist/cli.js`), peer dep on `@colbymchenry/codegraph`, publishes `dist/`, `hooks/`, `scripts/`
- `tsconfig.json` — Strict mode, commonjs modules, declarations + source maps, rootDir `src/` → outDir `dist/`
- `README.md` — User-facing docs (Quick Start, two-layer architecture, CLI reference)
- `CONTRIBUTING.md` — Dev setup, isolated testing via `CKG_CONFIG_DIR`, release process

## Architecture
Two-layer system:
1. **CodeGraph (Layer 1)** — SQLite symbol graph exposed as MCP server; registered in `~/.claude.json`
2. **CLAUDE.md (Layer 2)** — Per-directory narrative context generated by Anthropic API; kept current by Python hooks

## Conventions
- TypeScript strict mode — no `any` without explanatory comment
- All async functions require proper error handling
- Python hooks must never crash Claude Code — wrap in `try/except`, log to `~/.ckg/logs/`
- Deep-merge all JSON config files — never overwrite existing keys
- Comments explain *why*, never *what*
- Config: `~/.ckg/config.json` · Hooks: `~/.claude/hooks/` · MCP: `~/.claude.json`

## Development
```bash
npm install && npm run build   # compile TS
npm link && ckg --version      # test CLI locally
npm run dev                    # tsc watch mode
```

### Isolated Testing
```bash
export CKG_CONFIG_DIR=/tmp/ckg-test
node dist/cli.js install                    # redirects config writes
CKG_CONFIG_DIR=/tmp/ckg-test python3 scripts/init_claude_md.py --path /tmp/test-project --dry-run
```

### Testing Hooks Manually
```bash
echo '{"tool_name": "Write", "tool_input": {"file_path": "/tmp/test-project/src/foo.ts"}}' \
  | CKG_CONFIG_DIR=/tmp/ckg-test python3 hooks/post_tool_use.py
echo '{"stop_hook_active": false}' \
  | CKG_CONFIG_DIR=/tmp/ckg-test python3 hooks/stop.py
```

### Release
```bash
npm run build && npm version [patch|minor|major] && npm publish --access public
```
